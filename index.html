[FROM_DB_TO_DB APPLICATION]

Данная инструкция относится к пользователям,

которые захотели использовать данное приложение через командную строку.

Что необходимо сделать для запуска приложения из CMD.

  1. Создать файл с расширением .groovy
  2. Сохранить в нем следующие настройки:

data=
[
    "from":
    [
        "USER":"USERNAME",
        "PASS":"PASS",
        "CONNECTION_STRING":"jdbc:sqlserver://;servername=OTZ-prod1;integratedSecurity=true",
        "DRIVER":"com.microsoft.sqlserver.jdbc.SQLServerDriver",
        "QUERY":"select * from asddgasdgasdgasgd",
        "NAME_FIELD":"asdgsdg"
    ],
    
    "to":
    [
        "USER":"USERNAME",
        "PASS":"PASS",
        "BATCH_INSERT":5000,
        "CONNECTION_STRING":"jdbc:sqlserver://;servername=OTZ-prod1;integratedSecurity=true",
        "DRIVER":"com.microsoft.sqlserver.jdbc.SQLServerDriver",
        "QUERY":"insert into sdsdgsgd values (?, ?, ? asdsgsagasdgasdg)",
        "NAME_FIELD":"111111"
    ]
]

3. Важно! Не нужно менять структуру настроек.

Все, что от вас требуется:

- установить настройки для исходящего подключения БД

- установить настройки для входящего подключения БД

Важно! Все настройки должны быть в одну строку.

4. После того, как вы создали и заполнили файл настроек, его необходимо сохранить в удобном для Вас месте.

5. Создание bat файла для запуска файла настроек:

- создаете файл с расширением .bat

- указываете в нем:

echo off
%JAVA_HOME%\bin\java -jar "E:\_own_repository_data\groovy\trunk\from_db_to_db_gui-\dist\from_db_to_db_gui.jar" "C:\Users\ezhov_da\Desktop\groovy_test_reader.groovy"

где:

%JAVA_HOME%\bin\java  - путь к Java в Вашей системе, если в переменной окружения не настроена %JAVA_HOME%  - укажите полный путь,

например: C:\jre1.8.0_121_x86\bin\java.exe

"E:\_own_repository_data\groovy\trunk\from_db_to_db_gui-\dist\from_db_to_db_gui.jar"  - абсолютный путь к файлу приложения

"C:\Users\ezhov_da\Desktop\groovy_test_reader.groovy"  - абсолютный путь к файлу настроек

 

После чего сохраняете bat файл и можете запускать.

Весь лог выводится в файл fdtd.log папки приложения.
